# Developer → QA Handoff

## What Changed

| File | Action | Description |
|------|--------|-------------|
| `package.json` | Created | Project manifest with `start` script (`tsx src/index.ts`), `build` script (`tsc`), and devDeps: typescript, tsx, @types/node |
| `tsconfig.json` | Created | TypeScript config targeting ES2020/CommonJS for Node.js |
| `src/index.ts` | Created | Entry point — seeds a CPU baseline sample, then starts a `setInterval` poll every 2 s; handles SIGINT/SIGTERM cleanly |
| `src/metrics.ts` | Created | Collects CPU % (calculated from `os.cpus()` diff), memory used/total (via `os.totalmem()`/`os.freemem()`), system uptime (`os.uptime()`), and top-3 processes by CPU (`ps -axo pid,comm,%cpu` via `child_process.execSync`) |
| `src/render.ts` | Created | Draws the ANSI dashboard: clears screen each tick, renders CPU bar (green ≤60%, yellow >60%, red >80%), memory bar (same thresholds), uptime, and top-processes table |

## How to Test

```bash
npm install
npm run start
```

**Expected happy-path behavior:**
- Terminal clears every 2 seconds and redraws the dashboard
- CPU section shows a percentage and a filled progress bar in the correct color
- Memory section shows used/total MB with a bar
- Uptime section shows days/hours/minutes/seconds since system boot
- Top Processes table lists up to 3 processes with PID, name, and CPU%
- `Ctrl+C` exits cleanly without a dangling cursor

**Color thresholds to verify:**
- CPU ≤ 60% → green bar
- CPU 61–80% → yellow bar
- CPU > 80% → red bar
- Same thresholds apply independently to the memory usage bar

**Edge cases to test:**
- First render (before baseline) correctly shows 0% CPU — then subsequent ticks show actual usage
- `ps` output with long process paths — names are truncated at 22 chars
- System with very low memory headroom (>80% used) — memory bar should appear red

## Environment Setup

- **No new env vars needed**
- **Install:** `npm install` (or `npm ci`)
- **No database migrations**

## Notes

- The `ps` command used (`ps -axo pid,comm,%cpu`) is BSD-compatible (macOS + Linux)
- CPU measurement requires two consecutive samples; the very first render intentionally seeds the baseline without displaying. The first visible tick (at t=2s) shows real usage.
- `tsx` is a devDependency used only as the TypeScript runner for `npm run start`. There are zero runtime npm dependencies — all system data collection uses Node.js built-ins (`os`, `child_process`).
- `dist/` directory is generated by `npm run build` (tsc) — not needed at runtime when using `npm run start`
